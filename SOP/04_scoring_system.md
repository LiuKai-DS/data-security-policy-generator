# 策略评分体系参考

> **核心原则**：风险偏好是第一层过滤器（确定策略动作上限），分值计算是第二层（仅在"平衡"偏好下触发）。

---

## 第一层：风险偏好决策树

```
用户发起数据传输操作
        │
        ▼
┌─────────────────────────┐
│   风险偏好 = 激进？      │
└─────────┬───────────────┘
          │
    ┌─────┴─────┐
    │           │
   是          否
    │           │
    ▼           ▼
L4数据？   ┌─────────────────┐
    │     │ 风险偏好=平衡？  │
    │     └────────┬────────┘
    │         ┌───┴───┐
    │        是        否
    │         │         │
    │         ▼         ▼
    │    计算分值    风险偏好=保守
    │    查阈值      │
    │         ▼      ▼
    │    策略动作    最多审批放行
    ▼
 阻断（L4直断）
```

### 风险偏好与动作上限

| 风险偏好 | 允许的最高动作 | 核心逻辑 |
|---------|------------|--------|
| **激进** | **阻断** | L4核心商密直接阻断；其余按公式计算 |
| **平衡** | **阻断** | 按分值阈值执行（54/27/9分段） |
| **保守** | **审批放行** | 永远不阻断，弹窗视为审批，最高只到审批 |

### 激进模式特殊规则

当风险偏好=**激进**时：
- **L4核心商密 × 任意行为/影响 = 直接阻断**，无需计算分值
- 例：员工私发战略规划至个人邮箱 → L4数据，直接阻断，不管行为风险高低

## 第二层：分值计算（仅平衡偏好触发）

### 评分公式

```
策略分值 = 数据等级分值 × 行为风险分值 × 业务影响分值
```

> ⚠️ 注意：风险偏好不参与乘积计算，仅作为动作上限。

**分值范围**：1分（最低组合）~ 36分（最高组合，平衡偏好下）

### 各维度评分标准

#### 数据等级（客观维度）

| 级别 | 名称 | 分值 | 说明 |
|------|------|------|------|
| L4 | 核心商密 | **4分** | 泄露造成严重业务损失，如战略规划、核心技术、高管薪酬 |
| L3 | 普通商密 | **3分** | 泄露造成较大影响，如合同、客户信息、财务数据 |
| L2 | 内部知悉 | **2分** | 泄露造成一般影响，如内部通知、部门资料 |
| L1 | 外部公开 | **1分** | 泄露无实际影响，如公开发布的信息 |

#### 行为风险（客观维度）

| 级别 | 分值 | 典型场景 |
|------|------|---------|
| 高 | **3分** | 使用VPN/代理规避、批量下载、离职前异常访问、压缩加密外发 |
| 中 | **2分** | 非工作时间外发、非常用IM传输、私发个人邮箱 |
| 低 | **1分** | 通过公司邮箱向工作伙伴发送、打印敏感文件 |

#### 业务影响（主观维度）

| 级别 | 分值 | 判断标准 |
|------|------|---------|
| 高 | **3分** | 影响核心业务运营、涉及监管合规、可能引发法律风险 |
| 中 | **2分** | 影响部门正常工作、可能造成经济损失 |
| 低 | **1分** | 影响可控，有备份或补救措施 |

## 分值分布与动作对照（平衡偏好）

```
36分（最高） ──────────────────┐
                               │ ① 阻断（≥27分）
27分  ─────────────────────────┤
                               │
9分   ─────────────────────────┤ ② 审批放行（≥9分）
                               │
1分   ─────────────────────────┤ ③ 弹窗警告（1~8分）
                               │
1分（最低）────────────────────┤
```

| 得分区间 | 策略动作 | 典型组合示例 |
|---------|---------|------------|
| **27 ~ 36分** | **阻断** | L4×高×高(36) / L3×高×高(27) |
| **9 ~ 26分** | **审批放行** | L4×中×高(24) / L3×高×中(18) / L4×低×高(12) |
| **1 ~ 8分** | **弹窗警告** | L2×中×低(4) / L3×低×低(3) |
| — | **仅审计** | 建设初期默认，不参与分值计算 |

> ⚠️ **阈值修正说明**：平衡偏好下最高分值=4×3×3=36分（不含风险偏好乘数）。阻断阈值54不适用于修正后逻辑，改为：≥27分（75%）=阻断，≥9分（25%）=审批，1~8分=弹窗。

## 典型场景分值计算示例

### 激进偏好

| 场景 | 数据等级 | 行为风险 | 业务影响 | 结果 |
|------|---------|---------|---------|------|
| 员工私发战略规划至个人邮箱 | L4 | 高 | 高 | **L4直断** |
| 员工私发内部通知至个人邮箱 | L2 | 高 | 低 | 公式:2×3×1=6→弹窗 |
| 财务发送客户名单至合作伙伴 | L3 | 中 | 高 | 公式:3×2×3=18→审批 |

### 平衡偏好

| 场景 | 数据等级 | 行为风险 | 业务影响 | 总分 | 动作 |
|------|---------|---------|---------|------|------|
| 研发非工作时间批量下载源代码 | L4 | 高 | 高 | 36分 | **阻断** |
| 财务发送含客户的报表至合作伙伴 | L3 | 中 | 中 | 18分 | **审批放行** |
| 员工通过公司邮箱发公开资料 | L1 | 低 | 低 | 1分 | **弹窗警告** |
| 普通员工发送部门周报 | L2 | 低 | 低 | 2分 | **审批放行**（保守偏好下，弹窗视为审批，不阻断） |

### 保守偏好

| 场景 | 结果 |
|------|------|
| 任何L4核心商密外发 | **审批放行**（最高动作=审批，不阻断） |
| 任何L3普通商密外发 | **审批放行** |
| 低风险L1/L2数据外发 | **弹窗警告** |

## 三阶段数据安全建设与策略动作演进

| 阶段 | 适用情况 | 策略动作配置 |
|------|---------|------------|
| **前期（起步期）** | 刚部署DLP/UEBA，需先建基线 | **全部设为仅审计**，积累30天行为数据后再评估 |
| **中期（建设期）** | 已建立基线，开始精细化运营 | L4数据启用**弹窗警告**，L3以上启用**审批放行** |
| **后期（成熟期）** | 运营机制完善，按公式严格执行 | **按风险偏好+分值阈值**执行激进/平衡/保守三套逻辑 |

> 建设初期不建议开启阻断，以免因误报影响业务连续性。建议至少运行90天审计后再逐步升级动作级别。

## 特殊场景附加分规则

| 场景 | 附加规则 |
|------|---------|
| 离职员工（30天内） | 在行为风险维度+1分（封顶3分） |
| 核心商密外发至境外 | 直接封顶阻断，不受保守偏好约束 |
| 用户首次触发后主动纠正 | 在业务影响维度-1分（最低1分） |
| 有完整备份机制的业务 | 在业务影响维度-1分（最低1分） |
| 新入职员工（90天内） | 行为风险默认+1分（熟悉期异常检测） |
