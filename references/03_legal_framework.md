# 数据安全法律法规参考手册

> Skill使用说明：向用户收集信息时，可根据企业所属行业和风险偏好，有针对性地引用以下法规条款，作为策略设计的法律依据。策略说明中应体现「依据《XX法》第XX条」字样。

## 二、核心法规逐条解读

### 2.1 中华人民共和国网络安全法（2025年修订版，2026年1月1日施行）

**地位**：网络安全领域基础性法律，2017年6月1日首次施行，2025年11月修订后2026年1月1日起施行。

#### 重点条款

| 条款 | 内容摘要 | 与策略设计的关联 |
|------|---------|--------------|
| **第21条** | 国家实行网络安全等级保护制度，网络运营者应当按照网络安全等级保护制度的要求，履行安全保护义务 | 策略设计必须满足等保2.0的技术要求（边界防护、访问控制、审计日志等） |
| **第23条** | 关键信息基础设施运营者应当在网络安全等级保护制度的基础上，实行重点保护 | 央国企若为关键信息基础设施运营者，需在等保三级基础上额外增加防护措施 |
| **第27条** | 任何个人和组织不得从事非法侵入他人网络、干扰他人网络正常功能、窃取网络数据等危害网络安全的活动；不得提供专门用于侵入、干扰等危害网络安全活动的程序、工具 | 策略设计需检测外部入侵行为和恶意工具使用 |
| **第31条** | 国家对公共通信和信息服务、能源、交通、水利、金融、公共服务、电子政务等重要行业和领域，以及其他一旦遭到破坏、丧失功能或者数据泄露，可能严重危害国家安全、国计民生、公共利益的关键信息基础设施，在网络安全等级保护制度的基础上，实行重点保护 | 央国企总部多数属于关键信息基础设施保护范围 |
| **第34条** | 关键信息基础设施运营者应当设置专门安全管理机构，并对该机构负责人进行安全背景审查；采取数据分类、重要数据备份和加密等措施 | 企业必须建立数据分类分级制度，DLP策略与分类分级直接对应 |
| **第37条** | 关键信息基础设施运营者在中华人民共和国境内运营中收集和产生的个人信息和重要数据，**应当在境内存储**；因业务需要，确需向境外提供的，应当按照国家网信部门会同国务院有关部门制定的办法进行安全评估 | 数据出境场景必须触发阻断或审批策略 |

### 2.2 中华人民共和国数据安全法（2021年9月1日施行）

**地位**：数据安全领域基础性法律，建立数据分类分级保护制度。

#### 重点条款

| 条款 | 内容摘要 | 与策略设计的关联 |
|------|---------|--------------|
| **第6条** | 国家保护个人、组织与数据有关的权益 | 策略设计需体现对个人数据权益的保护 |
| **第21条** | 国家建立数据分类分级保护制度，根据数据在经济社会发展中的重要程度，以及一旦遭到篡改、破坏、泄露或者非法获取、非法利用，对国家安全、公共利益或者个人、组织合法权益造成的危害程度，进行数据分类分级 | **核心条款**：数据分类分级是策略设计的基础框架，本skill的L4/L3/L2/L1分级体系与此对应 |
| **第27条** | 开展数据处理活动应当依照法律、法规的规定，建立健全全流程数据安全管理制度，组织开展数据安全教育培训，采取相应的技术措施和其他必要措施，保障数据安全 | 企业需建立DLP/UEBA等技术措施，策略设计是合规的技术实现 |
| **第29条** | 重要数据的处理者应当明确数据安全负责人和管理机构，落实数据安全保护责任 | 策略设计需支持"数据安全责任人"审批流程 |
| **第31条** | 重要数据的出境安全管理，适用《网络安全法》的规定；其他数据处理者在中华人民共和国境内运营中收集和产生的重要数据的出境安全管理，适用国家网信部门会同国务院有关部门制定的办法 | 数据出境场景需作为独立策略类型设计 |
| **第32条** | 任何组织、个人收集数据，应当在享有数据处理法定权限的范围内进行 | 策略设计需检测超权限数据收集行为 |
| **第45条** | 违反本法规定，有下列情形之一的，责令改正，给予警告，可以并处罚款：拒不改正或者造成大量数据泄露等严重后果的，处50万元以上罚款 | 合规策略的设计和执行直接关系到企业行政处罚风险 |

### 2.3 中华人民共和国个人信息保护法（2021年11月1日施行）

**地位**：个人信息保护领域专门法律，对敏感个人信息和一般个人信息分别规定。

#### 重点条款

| 条款 | 内容摘要 | 与策略设计的关联 |
|------|---------|--------------|
| **第13条** | 个人信息处理者处理个人信息，应当在事先充分告知的前提下取得个人同意 | 策略设计需支持"告知-同意"机制 |
| **第28条** | 敏感个人信息是一旦泄露或者非法使用，可能导致人格尊严受到侵害或者人身、财产安全危险的个人信息，包括**医疗健康信息、金融账户信息、行踪轨迹、不满十四周岁未成年人的个人信息**等 | 敏感个人信息属于L3/L4级别，需重点防护 |
| **第29条** | 处理敏感个人信息应当取得个人的**单独同意** | 策略设计需检测未获单独同意的敏感信息处理行为 |
| **第51条** | 个人信息处理者应当根据个人信息的处理目的、处理方式、个人信息的种类以及对个人权益的影响、可能存在的安全风险，采取加密、去标识化等安全技术措施 | 策略设计中的"加密检测"和"去标识化检测"与此对应 |
| **第55条** | 处理敏感个人信息、利用个人信息进行自动化决策等高风险活动，应当事前进行个人信息保护影响评估 | 策略设计可作为影响评估的技术支撑 |
| **第69条** | 处理个人信息侵害个人信息权益造成损害，个人信息处理者不能证明自己没有过错的，应当承担损害赔偿等侵权责任 | 策略缺失或失效可能导致企业承担连带责任 |
| **第66条** | 违反本法规定处理个人信息，或者处理个人信息未履行本法规定的个人信息保护义务的，情节严重的，处5000万元以下或者上一年度营业额5%以下罚款 | 最高罚款可达营业额5%，策略设计合规性至关重要 |

### 2.4 网络数据安全管理条例（国务院令第790号，2025年1月1日施行）

**地位**：衔接《网络安全法》《数据安全法》《个人信息保护法》的实施细则。

#### 重点条款

| 条款 | 内容摘要 | 与策略设计的关联 |
|------|---------|--------------|
| **第10条** | 网络数据处理者提供的网络产品、服务应当符合相关国家标准的强制性要求；发现安全缺陷、漏洞等风险时，应当立即采取补救措施 | 策略设计需包含"安全漏洞/补丁检测"类策略 |
| **第11条** | 网络数据处理者应当建立完善的安全管理制度，包括数据分类分级管理、安全监测预警、应急处置等 | 企业必须建立分级管理机制，策略表是具体执行工具 |
| **第15条** | 重要数据的处理者应当明确数据安全负责人，并报所在地省级网信部门 | 策略审批流程需对接"数据安全负责人"角色 |
| **第23条** | 网络数据处理者委托处理个人信息的，应当与受托人约定处理目的、期限、方式、个人信息种类、保护措施以及双方权利义务，并对受托人的个人信息处理活动进行监督 | 第三方数据处理场景需设计监控策略 |
| **第27条** | 网络数据处理者向境外提供个人信息和重要数据的，应当依法开展数据出境安全评估 | 数据出境策略必须包含安全评估前置步骤 |
| **第44条** | 重要数据处理者应当自行或者委托数据安全服务机构每年开展一次数据安全风险评估 | 策略有效性评估可纳入年度风险评估 |

### 2.5 关键信息基础设施安全保护条例（国务院令第745号，2021年9月1日施行）

**地位**：对《网络安全法》第31条的细化，明确关基运营者的安全保护义务。

#### 重点条款

| 条款 | 内容摘要 | 与策略设计的关联 |
|------|---------|--------------|
| **第4条** | 关键信息基础设施，是指公共通信和信息服务、能源、交通、水利、金融、公共服务、电子政务、国防科技工业等重要行业和领域的，以及其他一旦遭到破坏、丧失功能或者数据泄露，可能严重危害国家安全、国计民生、公共利益的重要网络设施、信息系统等 | 央国企总部大概率属于关基范围，需执行强化策略 |
| **第6条** | 运营者应当建立健全网络安全保护制度和责任制，设置专门安全管理机构 | 策略设计需支持组织架构对应（安全机构→审批流程） |
| **第12条** | 运营者应当自行或者委托网络安全服务机构对关键信息基础设施每年至少进行一次网络安全检测和风险评估 | 策略有效性评估是合规要求的一部分 |
| **第17条** | 运营者采购网络产品和服务，网络安全服务机构提供服务等，应当按照规定与提供者签订安全保密协议，明确安全和保密义务与责任 | 第三方访问场景需设计监控和限制策略 |
| **第21条** | 运营者应当设置冗余备份，建立灾难备份，恢复措施 | 策略设计需考虑业务连续性，避免过度阻断影响业务 |

### 2.6 网络安全审查办法（2022年2月15日施行）

**地位**：对影响或可能影响国家安全的网络数据处理活动进行审查。

#### 重点条款

| 条款 | 内容摘要 | 与策略设计的关联 |
|------|---------|--------------|
| **第7条** | 掌握超过100万用户个人信息的网络平台运营者赴国外上市，必须向网络安全审查办公室申报网络安全审查 | 涉及大规模用户数据的策略设计需考虑上市合规路径 |
| **第10条** | 网络安全审查重点评估采购活动、数据处理活动以及国外上市对国家安全的影响 | 策略设计中的"境外传输检测"与审查要求对应 |

## 四、法律责任汇总

### 4.1 处罚类型一览

| 法律 | 最高处罚 | 适用情形 |
|------|---------|---------|
| 《网络安全法》 | 100万元罚款；情节严重者100万元+停业整顿+吊销执照 | 网络安全保护义务未履行 |
| 《数据安全法》 | 50万元罚款（一般）；1000万元罚款（情节严重） | 数据安全保护义务未履行 |
| 《个人信息保护法》 | 5000万元罚款或上一年度营业额5%（情节严重） | 个人信息处理违规 |
| 《网络数据安全管理条例》 | 100万元罚款（一般）；500万元罚款（情节严重） | 网络数据安全管理义务未履行 |
| 《保守国家秘密法》 | 构成刑事犯罪，依法追究刑事责任 | 国家秘密泄露 |

### 4.2 刑事风险

| 罪名 | 适用情形 | 法定刑 |
|------|---------|--------|
| **侵犯公民个人信息罪** | 违反国家有关规定，向他人出售或者提供公民个人信息 | 情节严重：三年以下有期徒刑；情节特别严重：三年以上七年以下有期徒刑 |
| **非法获取计算机信息系统数据罪** | 侵入前述系统以外的其他计算机信息系统获取数据 | 三年以下有期徒刑或者拘役，并处或者单处罚金 |
| **为境外窃取、刺探、收买、非法提供商业秘密罪** | 为境外的机构、组织、人员窃取、刺探、收买、非法提供商业秘密 | 五年以下有期徒刑；情节严重：五年以上有期徒刑 |

## 五、策略设计的法规映射

### 5.1 策略类型 → 法规依据

| 策略类型 | 主要法规依据 |
|---------|------------|
| **规则型（R00X）** | 《网络安全法》第21条（等保）、第27条（禁止危害网络安全的程序） |
| **限制型（L00X）** | 《数据安全法》第21条（分类分级）、第27条（安全管理制度） |
| **访问型（A00X）** | 《网络安全法》第34条（访问控制）、第21条（身份鉴别） |
| **内容型（C00X）** | 《个人信息保护法》第28-29条（敏感个人信息）、《数据安全法》第32条（合法收集） |
| **上下文型（CT00X）** | 《数据安全法》第21条（危害程度评估）、第31条（出境管理） |
| **敏感行为型（S00X）** | 《网络安全法》第27条（危害网络安全活动）、《数据安全法》第32条（超权限收集） |

### 5.2 策略动作 → 法规强制程度

| 策略动作 | 法规强制程度 | 说明 |
|---------|-----------|------|
| **阻断** | 强制性合规要求 | 法律明确禁止的行为（如数据出境未评估）必须阻断 |
| **审批放行** | 法定程序要求 | 重要数据处理、重要操作需经数据安全责任人审批 |
| **弹窗警告** | 告知义务履行 | 处理敏感个人信息前需取得单独同意，弹窗是技术实现 |
| **仅审计** | 记录保存义务 | 《网络安全法》等要求保存网络日志不少于六个月 |

### 5.3 行业×法规优先级

| 行业 | 最高优先级法规 | 核心合规重点 |
|------|------------|-----------|
| **央国企总部** | 《网络安全法》+《关键信息基础设施保护条例》+《保守国家秘密法》 | 等保三级+关基保护+国家秘密管理 |
| **金融机构** | 《网络安全法》+《数据安全法》+JR/T 0197 | 等保+金融数据分级+监管报送 |
| **互联网/科技** | 《个人信息保护法》+《数据安全法》+《网络数据安全管理条例》 | 个人信息保护+数据出境合规+自动化决策告知 |
| **医疗健康** | 《个人信息保护法》+《数据安全法》+GB/T 45574 | 患者隐私保护+医疗数据分级 |
| **一般企业** | 《网络安全法》+《数据安全法》 | 基础等保+数据分类分级 |

## 六、使用建议

1. **向用户收集信息时**：根据企业行业类型，优先引用对应的法规重点条款
2. **策略说明中**：标注法规依据，如"依据《数据安全法》第21条，数据分类分级是法定义务"
